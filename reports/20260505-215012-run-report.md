# 20260505-215012 Benchmark 报告

## 范围

- Run: `20260505-215012`
- Template 集合: `templates/fill-grid/site-6x6`
- 模型:
  - `mimo-v2.5-pro`
  - `deepseek-v4-flash`
  - `deepseek-v4-pro`
  - `gpt-5.4`

本次结果全部来自 `site-6x6` 的 20 个模板。

## 总结

这次 run 的结论很清楚：

1. `mimo-v2.5-pro` 是综合最强。
2. `deepseek-v4-pro` 在少数模板上能拿到单题第一，但整体被时间分明显拖累。
3. `deepseek-v4-flash` 是稳定的快速 baseline。
4. `gpt-5.4` 合法性没问题，但速度和多题多样性都不占优，最终分落后。

另外一个很重要的观察是：

- 四个模型的 `validPuzzleRate` 都是 `1.0`
- `preferenceFit` 也都是 `1.0`

所以这次真正拉开差距的，主要是：

- `crossPuzzleVariety`
- `elapsedMs / timeScore`

换句话说，这次 run 更像是在比较：

- 多题去重质量
- 求解速度

而不是在比较“能不能做对”。

## 排名解读

### 1. mimo-v2.5-pro

优点：

- 平均 `finalScore` 最高
- 平均 `overallScore` 也最高
- 平均耗时最低
- `crossPuzzleVariety` 也是四者里最强

这意味着它不是单纯“靠快取胜”，而是：

- 内容分本身高
- 时间也明显领先

本次 20 个模板里，它拿了 `13` 个单题第一。

### 2. deepseek-v4-flash

特点：

- 很快
- 合法性稳定
- 整体内容分不低

它的核心定位还是很清晰：

- 作为快速 baseline 很好
- 但在 `crossPuzzleVariety` 上略弱于 `mimo-v2.5-pro`
- 所以即使很快，最终分仍有明显差距

### 3. deepseek-v4-pro

特点：

- 少数模板上能打赢所有模型
- `overallScore` 很高
- `crossPuzzleVariety` 也不错

问题：

- 时间代价太大
- 平均耗时约 `2284ms`，比 `mimo-v2.5-pro` 高很多

它在 20 个模板里拿了 `7` 个单题第一，说明：

- 求解质量是有竞争力的
- 但当前实现的速度成本过重

### 4. gpt-5.4

特点：

- 合法率没有问题
- `overallScore` 仍然高

问题：

- 平均耗时明显偏高
- `crossPuzzleVariety` 是四者里最低

所以它这次的问题不是“做不出来”，而是：

- 多题结果相似度更高
- 速度也没有优势

## 对评分函数的观察

这次 run 有一个非常明显的现象：

- 所有模型 `validPuzzleRate = 1`
- 所有模型 `preferenceFit = 1`
- `overallScore` 彼此非常接近

因此：

- `finalScore` 的主要区分度来自 `timeScore`
- 次要差异来自 `crossPuzzleVariety`

这说明当前评分函数在这组模板上，更擅长区分：

- 速度
- 多题去重

而不太能区分：

- 基础合法性
- 偏好满足

这不是错误，但意味着：

- 这组 `site-6x6` 模板对“正确性”区分度不够
- 更适合测“性能 + 多样性”

## 结论

如果按这次 run 的最终结果看：

1. `mimo-v2.5-pro` 是当前最强实现  
   优势来自：内容分高、速度快、多样性也强。

2. `deepseek-v4-pro` 是最有潜力的质量型实现  
   但速度代价太重，当前 benchmark 下吃亏明显。

3. `deepseek-v4-flash` 是很稳的快速 baseline  
   不如 `mimo-v2.5-pro` 强，但工程上很实用。

4. `gpt-5.4` 当前主要问题不是合法性，而是：  
   多题结果更相似，且性能不占优。

## 建议

- 如果目标是挑当前最佳默认实现：选 `mimo-v2.5-pro`
- 如果目标是继续优化一个“高质量路线”：优先优化 `deepseek-v4-pro` 的搜索性能
- 如果目标是保留一个快速对照 baseline：保留 `deepseek-v4-flash`
- 如果目标是继续提升 `gpt-5.4`：
  - 先看 `page-019 / page-004 / page-016`
  - 重点优化多题结果选择与重复控制
